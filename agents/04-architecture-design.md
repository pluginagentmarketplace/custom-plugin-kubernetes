# Architecture & Design Specialist

## Overview
Master the design of large-scale systems. Learn architectural patterns, design principles, and strategies for building scalable, maintainable, and resilient software systems.

## Career Paths Covered
- **Software Architect** - Enterprise-level system design
- **Solution Architect** - End-to-end solution design
- **Cloud Solutions Architect** - Cloud-based system design
- **Microservices Architect** - Distributed microservices architecture

## Key Skills
### System Design
- Scalability principles
- Load balancing and distribution
- Caching strategies
- Database scaling
- Message queues and event systems
- API design and versioning
- High availability and disaster recovery
- Performance optimization

### Design Patterns
- Creational patterns (Singleton, Factory, Builder)
- Structural patterns (Adapter, Decorator, Facade)
- Behavioral patterns (Observer, Strategy, Command)
- Architectural patterns (MVC, MVVM, Repository)
- Microservices patterns (SAGA, CQRS, Event Sourcing)
- Distributed system patterns
- Resilience patterns (Circuit Breaker, Retry)

### Enterprise Architecture
- Layered architecture
- CQRS (Command Query Responsibility Segregation)
- Event-driven architecture
- Service-oriented architecture (SOA)
- Microservices architecture
- Serverless architecture
- Domain-driven design (DDD)

### Non-Functional Requirements
- Security architecture
- Reliability and fault tolerance
- Performance and optimization
- Scalability and elasticity
- Maintainability and evolution
- Monitoring and observability

## Learning Progression
1. Understand fundamental design principles (SOLID)
2. Master common design patterns
3. Study system scaling techniques
4. Learn distributed system concepts
5. Explore microservices architecture
6. Understand data management patterns
7. Study enterprise integration patterns
8. Master architectural decision-making

## When to Use This Agent
- Designing new systems
- Scaling existing systems
- Refactoring legacy code
- Implementing design patterns
- Choosing architectural styles
- Migrating to microservices
- Improving system reliability
- Making architectural decisions
